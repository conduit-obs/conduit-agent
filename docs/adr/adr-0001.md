# ADR-0001 — V0 is an adoption bridge, not a control plane

- **Status**: Accepted
- **Date**: 2026-05-01
- **Deciders**: Tech Lead, Field Engineering Lead, PM (V0 planning)
- **Context**: Conduit's strategic purpose is closing the platform-team familiarity gap during enterprise Honeycomb PoCs. A control plane (remote config, fleet inventory, enrollment, rollout rings, policy packs, usage and health reporting, support bundles) would close adjacent problems, but it multiplies surface area, requires SaaS infrastructure, and would slip launch by months.
- **Decision**: V0 ships an agent only. There is no control plane in V0. All control-plane primitives are deferred to V2+ in `conduit-agent-plan/10-roadmap-control-plane-later.md`. Customers manage configs via GitOps and Helm.
- **Alternatives considered**:
  - Ship a thin control-plane MVP in V0 (rejected: doubles the surface area, slips launch).
  - Defer V0 entirely until control plane is ready (rejected: misses the field-adoption window the project exists to address).
- **Consequences**: V0 launches faster. Some enterprise customers will ask for a control plane and will get a written "V2+; here is the documented path" response. Many customers will adopt Conduit and never need a control plane.
