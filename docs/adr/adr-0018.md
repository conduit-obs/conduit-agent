# ADR-0018 — Demo script is a first-class V0 deliverable

- **Status**: Accepted
- **Date**: 2026-05-01
- **Deciders**: Field Engineering Lead, PM, DevRel (V0 planning)
- **Context**: Conduit's strategic purpose is field-led adoption. Without a repeatable demo, every field engineer reinvents the motion in each account — recreating exactly the bespoke-YAML problem that motivated Conduit in the first place. Documentation alone does not close this gap because the order, framing, and "what not to say" matter as much as the mechanics.
- **Decision**: The 30-minute field-engineer-runnable demo is a V0 release deliverable. It lives in [`conduit-agent-plan/11-demo-and-poc-script.md`](../../conduit-agent-plan/11-demo-and-poc-script.md), is rehearsed before launch, and the M13 milestone exits only when the rehearsal completes within the 30-minute time budget across at least three field engineers running independently.
- **Alternatives considered**:
  - Treat the demo as a post-launch nice-to-have (rejected: the strategic thesis depends on field-led adoption being repeatable from day one).
  - Document the demo without rehearsal (rejected: the time budget and recovery moments cannot be validated by reading).
- **Consequences**: M13 (V0 launch checklist) explicitly includes demo rehearsal as exit criterion. Field engineers' calendars need to absorb three rehearsal cycles before launch. Demo updates accompany every MINOR release that changes a customer-visible install or behavior.
