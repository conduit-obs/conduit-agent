# ADR-0011 — `conduit` CLI shape

- **Status**: Accepted
- **Date**: 2026-05-01
- **Deciders**: Tech Lead, PM (V0 planning)
- **Context**: Familiarity is one of the project's central design constraints. Platform engineers expect a single binary with a small subcommand surface, similar to Datadog Agent and Splunk UF.
- **Decision**: A single `conduit` binary with Cobra subcommands: `run`, `doctor`, `preview`, `config`, `version`, `send-test-data`. No per-platform CLI variants. Platform-specific differences live in flags (e.g., service-installation flags on Windows) and not in a separate binary.
- **Alternatives considered**:
  - Separate `conduitd` daemon and `conduitctl` control CLI (rejected: doubles the install surface for no V0 benefit).
  - Make `run` the implicit default with no subcommand (rejected: scriptability and discoverability suffer).
- **Consequences**: The CLI shape is small and learnable. New CLI behavior in V1+ slots into either an existing subcommand or a new top-level subcommand without breaking the mental model.
