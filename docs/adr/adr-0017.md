# ADR-0017 — JSON Schema generated from Go struct tags; CI enforces parity

- **Status**: Accepted
- **Date**: 2026-05-01
- **Deciders**: Tech Lead, PM (V0 planning)
- **Context**: Schema drift between code and docs is a chronic OSS-project failure mode. Hand-maintaining a JSON Schema alongside the Go struct hierarchy guarantees drift; CI parity checks pull the cost forward.
- **Decision**: A JSON Schema is generated from the `AgentConfig` Go struct tags and shipped with each release. CI fails if the generated schema is out of sync with the docs reference page. The generator becomes a build-time dependency.
- **Alternatives considered**:
  - Hand-maintained JSON Schema (rejected: drift).
  - No JSON Schema (rejected: customers and IDEs both want it).
  - Generate at runtime instead of at build (rejected: makes the docs CI gate harder to enforce).
- **Consequences**: Engineers cannot land a new config field without the generator picking it up. The schema is itself versioned (per [`conduit-agent-plan/08-release-and-support-model.md`](../../conduit-agent-plan/08-release-and-support-model.md) §"Versioning the JSON Schema") so customers can validate offline.
