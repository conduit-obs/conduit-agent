# ADR-0013 — Apache-2.0 license; clean-room implementation; Observe is reference only

- **Status**: Accepted
- **Date**: 2026-05-01
- **Deciders**: Tech Lead, Field Engineering Lead, Legal (V0 planning)
- **Context**: The Observe Agent (`github.com/observeinc/observe-agent`) is a public Apache-2.0 OpenTelemetry-Collector-wrapping agent. Its existence validates the market hypothesis Conduit operates on. Its source is legally available to copy. Three risks argue against doing so: positioning ("Conduit is a fork of Observe's agent" is a story we cannot afford), maintenance independence (forking ties our release pipeline to Observe's choices forever), and field credibility (customers running `git log` on Conduit and seeing Observe contributors will ask questions we don't want to answer in a sales conversation).
- **Decision**: Conduit is licensed under Apache-2.0. The Observe Agent is reference material only — patterns may be borrowed conceptually with attribution; **no code is copied verbatim**. Any pull request with verbatim copy from Observe is rejected and rewritten.
- **Alternatives considered**:
  - Fork Observe's agent (rejected: positioning, maintenance, field-credibility risks).
  - Copy Observe code with attribution (rejected: same).
  - Pick a different OSS license (rejected: Apache-2.0 is standard for OpenTelemetry-ecosystem projects and aligns with field-buyer expectations).
- **Consequences**: Implementation is slower than copy-and-modify. Positioning and maintenance independence are stronger. Attribution is honest and limited to reference docs ([`conduit-agent-plan/12-observe-reference-notes.md`](../../conduit-agent-plan/12-observe-reference-notes.md)) and ADRs.
