# ADR-0009 — TLS required by default for `output.mode: gateway`

- **Status**: Accepted
- **Date**: 2026-05-01
- **Deciders**: Tech Lead, Security (V0 planning)
- **Context**: Plaintext OTLP between an agent and a gateway leaks customer telemetry over the wire. Some lab and dev gateways do not have TLS; we must support those without making them the default.
- **Decision**: When `output.mode: gateway` is configured, TLS is required by default. `output.gateway.insecure: true` is honored as an explicit opt-out for plaintext, but `conduit doctor` issues a warning even when validation otherwise passes. TLS to `api.honeycomb.io` is non-overridable.
- **Alternatives considered**:
  - Plaintext-OK-by-default (rejected: insecure default; risk R-Sec-02).
  - Refuse plaintext entirely (rejected: lab and on-prem dev environments cannot configure their gateways to terminate TLS).
- **Consequences**: A misconfigured gateway endpoint without proper TLS material fails with a clear error. Customers using plaintext labs see warnings reinforcing that they should not ship that config to production. The doctor warning is the auditable record.
